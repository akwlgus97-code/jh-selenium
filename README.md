# Selenium QA Automation

Selenium을 활용해 웹 서비스의 핵심 사용자 흐름과 페이지 상태를 자동으로 검증하는 간단한 QA 자동화 예제입니다.

## 포함된 스크립트

- `kream_login.py`: KREAM 개인 회원 일반 로그인 자동화
- `page_health_check.py`: 네이버 주요 페이지 헬스체크 자동화

## 테스트 시나리오

### 1. 로그인 자동화

1. KREAM 홈페이지에 접속합니다.
2. 상단 로그인 링크를 클릭해 로그인 페이지로 이동합니다.
3. 이메일과 비밀번호를 입력합니다.
4. 로그인 버튼을 클릭합니다.
5. 로그인 후 상단 메뉴가 `로그아웃` 상태로 변경되는지 확인합니다.

### 2. 페이지 헬스체크

1. 네이버 사이트맵 페이지에 접속합니다.
2. 지정한 메뉴들을 클릭해 각 목적지 페이지로 이동합니다.
3. 응답 상태값이 `200`인지 확인합니다.
4. 랜딩 URL이 예상값과 일치하는지 확인합니다.

## 사용 기술

- Python
- Selenium WebDriver
- Chrome WebDriver
- WebDriverWait
- requests

## 참고 사항

- `kream_login.py`에는 실제 계정 정보가 포함되어 있지 않습니다.
- 현재 이메일/비밀번호 입력값은 빈 문자열로 되어 있으므로, 실제 실행 검증이 필요한 경우 테스트 전용 계정을 별도로 설정해야 합니다.
- 실무 환경에서는 계정 정보를 코드에 직접 작성하지 않고 환경변수나 설정 파일로 분리합니다.
